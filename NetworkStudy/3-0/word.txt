・IP
　　　Internet Protocolの略、データ通信の方法を定めた規約

・IPヘッダ
　　　パケットの先頭にある一定の形式に従って制御情報などを記載した部分

・ICMP
　　　Internet Control Message Protocolの略、IPによる通信で発生したエラー
　　　の通知や通信相手の動作確認、通信制御用メッセージのやり取りに使用さ
　　　れるプロトコル

・ICMPメッセージ
　　　問い合わせやエラーに関するメッセージをやり取りすることで、インター
　　　ネット通信においてIP通信の判定、フロー制御、Ping要求、応答、時間経
　　　過、Redirect機能、アドレスマスク要求、応答などの役割をする

・IPアドレス
　　　インターネットで通信するための住所、各機器に割り当てられ、通信相手
　　　を指定する際に使用。

　オクテット
　　　　情報量の単位の一つ、8ビットのこと
　　　　
　ネットワーク部
　　　　IPアドレスが属しているネットワークを識別するための部分

　ホスト部
　　　　ネットワーク内のコンピュータを識別するための部分

　クラス
　　　　ネットワークアドレスとホストアドレスを区切る際、規模別に分類した物

　プライベートIPアドレス
　　　　インターネットに直接接続しないコンピュータに割り当てられるIPアドレス

　グローバルIPアドレス
　　　　ICANNが管理している使用範囲が決められているIPアドレス

ネットワークアドレス
　　　ネットワーク部のアドレス

ブロードキャストアドレス
　　　ホスト部のビットが全て1のアドレス
　　　セグメントに接続されているすべてのホストにパケットを送信するための
　　　アドレス

ループバックアドレス
　　　IPアドレスの中で自らを指すもの、主に使用されるのは127.0.0.1


サブネット
　　　大規模なネットワーク内の小さなネットワーク
　　　IPネットワークを論理的に細分化したもの

サブネットマスク
　　　ネットワークの範囲を定義するために使用。サブネットマスクはIPv4のIP
　　　アドレスを使用する際にのみ使用

プレフィックス
　　　サブネットマスクの桁数を数字で表したもの

デフォルトゲートウェイ
　　　家庭や社内でのLANからインターネットなどのが言うネットワークに接続
　　　する際に、通信の出入り口の役割を果たす仕組みの事。
　　　
IPv6
　　　枯渇しないアドレス、めっちゃ割り当てられる

サブネットプレフィックス
　　　IPv4におけるサブネット、だいたい６４
　　　ネットワーク部にあたる部分

インターフェイスID
　　　ホスト部にあたる部分

コンソール接続　
　　　ルーターに設定を送るためにPCと接続するもの


VTY接続
　　　virtual teletypeの略　ルータなどの通信機器にネットワークを通じて
　　　別のコンピュータから接続し、操作や管理を行うための仮想的な接続口

AUX接続
　　　Auxiliary、予備や補助　が語源
　　　外部機器を接続し、音声を入出力するための端子

ルーティングテーブル
　　　ルータやネットワーク接続されてコンピュータが持つ、個々のネットワ
　　　－クの宛先への経路の一覧を保持しているテーブル状のデータ構造

ネクストホップ　
　　　ルータなどが持つ経路表に記載される情報の一つ、個々の宛先
　　　ネットワーク毎に決められた、次に点雄すべき隣接ルータのIPアドレス
　　　のこと

ARPテーブル
　　　通信を行う度に毎回問い合わせを行うのは非効率なため、一度ARPで
　　　調べたデータをOSなどが専用の一時的な記憶領域に保管しておき
　　　次に必要になったらそこから調べる仕組み、この対応表の事を言う

ARP
　　　Adress Resolution Protocol
　　　IPアドレスからネットワーク上のMACアドレスを求めるためのプロトコ
　　　ル、アープと呼ばれる、イーサネット上の機器間の物理的な通信
　　　では、IPアドレスではなく、機器固有のMACアドレスという物理アドレ
　　　スを用いる

コアルータ
　　　通信キャリアやISPネットワークの通信を中継するために利用される
　　　ルータを指し、大量のネットワークトラフィックを転送する必要がある
　　　ため、高速なパケット転送能力と高い可用性が求められる

センタールータ
　　　企業の拠点間WANネットワークで本社拠点、DC等、企業で中心的に利用
　　　されるネットワークを接続するために利用される。

エッジルータ
　　　企業の拠点間WANネットワークや支社や営業所等の拠点ネットワークを
　　　接続するために利用される。

ブロードバンドルータ
　　　家庭環境やSOHO環境でインターネット回線に接続する目的で利用される


ルーティング
　　　パケットのアドレス情報を読み取り、そのアドレスを基に
　　　ルーティングを行い、次に転送すべき最適な経路を決定


直接接続
　　　ルータと隣接しているネットワークを自動で登録する方法

スタティックルーティング
　　　ルータの隣接していないネットワークを手動で追加する方法

ダイナミックルーティング
　　　ルータの隣接していないネットワークについて、ルーティング
　　　プロトコルを使用して動的に追加する方法

ディスタンスベクタ型
　　　距離と方向により最適なルートを決定するアルゴリズム
　　　ポップ数で最適なルートを決定する。
　　　単純な動作であり、収束が遅いのが特徴

リンクステート型
　　　どのルータ同士が接続しているかという接続情報
　　　リンクステートを交換し合い、この情報の集合に基づいて経路を
　　　選択する方法。

ハイブリッド型
　　　上記二つの特性を兼ね備えたプロトコル
　　　Cisco独自のプロトコルのため、他社製のルータが混在する環境
　　　では使用できない。

トランスポート層
　　　上位の階層（アプリケーション層）からの要求を受けてデータを
　　　送出し、あるいは受信したデータを報告する。
　　　また下位の階層（インターネット層やネットワーク層）へデータ
　　　送信を要求し、受信したデータを受領する。
　　　伝送途上で参照、改変される事は稀である。
　　　主な役割はエラー検出、訂正と再送制御、コネクションの確立
　　　データの並び順の整列、フロー制御、輻輳制御
　　　applicationの識別など。

コネクション型通信
　　　データ通信において、通信を開始する前に相手との間で仮想的な
　　　専門通信路（こねくしょん）を確立し、それを通じてデータの
　　　送受信を行う通信方法

ACK
　　　ACKとは、二者間の通信で、相手方に何らかの肯定的な応答を返す際
　　　に送られる信号やデータ、パケットなどの事、
　　　接続要求を受理した、データが正しく受信できたといった内容
　　　を伝達するために用いられる

コネクションレス型通信
　　　コネクション型通信と異なり、事前にコネクションの確立を行わない
　　　

TCP
　　　1対1のセッションによる信頼性の高い通信を行うためのプロトコル
　　　Transmission Control Protocolの略
　　　パケットの再送やエラー訂正などを行う機能を持っているため、
　　　確実性をもった通信を行いたい場面で利用される。

スリーウェイハンドシェイク
　　　データ転送を行う前のコネクションの確立のことを指す
　　　接続を開始するか、許可と接続を開始していいか、それの了承
　　　による3段階の認証をさす

Telnet
　　　IPネットワークにおいて、遠隔地にあるサーバやルータ等を端末
　　　から操作する通信プロトコル、またそのプロトコルを利用する
　　　ソフトウェア

HTTP
　　　Hypertext Transfer Protocolの略
　　　WebサーバとWebブラウザの間で、Web情報をやりとりするための
　　　プロトコル、1つの要求に1つの応答を返す、例外はない

FTP
　　　ファイル転送を行うプロトコルの一種、
　　　Webサーバーにデータを転送する際に使われる、HTTPと違いポート
　　　が2種類あるため、細かい制御が可能
　　　File transfer Protocol

SMTP
　　　Simple Mail Transfer Protocol
　　　メール送信に関するProtocol、

TCPヘッダ
　　　データ転送単位であるTCPセグメントの先頭にある、制御情報
　　　が記載された部分の事、内容が固定されている部分が20バイト
　　　オプションが4バイト単位の可変長

UDP
　　　インターネット上で標準的に利用されている通信プロトコルの一種
　　　リアルタイム性が重視される音声通話や動画配信などで利用
　　　User Datagram Protocol

RTP
　　　real-time Transport Protocol
　　　音声や動画などのデータストリームをリアルタイムに配送するため
　　　のデータ通信プロトコル
　　　UDPと異なり、シーケンス番号がないため順序の組み立てができない
　　　が、タイムスタンプ（時刻情報）から各パケットの時間的な
　　　順序関係を把握し、データを再生する事ができる

SNMP
　　　Simple Network Management Protocol
　　　UDP/IPベースのネットワーク監視、ネットワーク管理を行う
　　　ためのプロトコル、ルータ、スイッチなどのネットワーク機器
　　　WindowsやUNIXサーバーなどの状態監視、リソース監視、
　　　パフォーマンス監視、トラフィック監視を行うために使用
　　　　　
DHCP
　　　Dynamic Host Configuration Protocolの略
　　　IPv4ネットワークにおいて通信用の基本的な設定を自動的に行う
　　　IPアドレスやサブネットマスク、デフォルトゲートウェイのIP
　　　アドレスなど

TFTP
　　　Trival File Transfer Protocol
　　　UDPを用いてコンピュータ間でファイルを転送するためのプロトコル
　　　効率はいいが信頼性に欠ける

RIP
　　　Routing Information Protocol
　　　AS内部における経路情報を交換するために用いられる距離ベクトル型
　　　のルーティングプロトコル
　　　Autonomous System

UDPヘッダ
　　　UDPのデータグラムの先頭部にある、制御情報などを一定の形式
　　　で記述した領域、その後ろに送りたいデータ本体を加えて、
　　　一つのUDPデータグラムとなる。
